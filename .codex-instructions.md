## Migration rules

1. `.get()` on SELECT queries:
   FROM: `const row = await db.select().from(table).where(eq).get()`
   TO:   `const [row] = await db.select().from(table).where(eq).limit(1)`
   (Remove `.get()`, add `.limit(1)`, destructure first element)

2. `.get()` on INSERT with RETURNING:
   FROM: `const row = await db.insert(table).values({}).returning().get()`
   TO:   `const [row] = await db.insert(table).values({}).returning()`
   (Remove `.get()`, destructure first element. `.run()` stays as-is.)

3. `.run()` on INSERT/UPDATE/DELETE: NO CHANGE. Already async-compatible.

4. Raw SQL `.all()`: ONLY in analytics/page.tsx (2 occurrences)
   FROM: `db.all<{...}>(sql\`SELECT...\`)`
   TO:   `const result = await db.execute(sql\`SELECT...\`); result.rows as unknown as {...}[]`

5. Raw SQL `.get()`: If exists:
   FROM: `db.get<{...}>(sql\`SELECT...\`)`
   TO:   `const result = await db.execute(sql\`SELECT...\`); result.rows[0] as unknown as {...}`

6. Transactions: NO CHANGE needed. `await db.transaction(async (tx) => { ... })` stays.
   But `.get()` inside tx also follows rule 1.

### 1.1 HIGH-SIGNAL FIND: Institutional-Grade Statistical Arbitrage Engine (Python)
**Repo:** `sharathStack/Statistical-Arbitrage-Pairs-Trading-Engine` ⭐1 | Updated Apr 6, 2026
**URL:** https://github.com/sharathStack/Statistical-Arbitrage-Pairs-Trading-Engine

**Why it matters:** This is the most complete open-source stat-arb implementation I've seen. It implements the full institutional pipeline end-to-end, not just cointegration + z-score.

**Pipeline:**
```
Universe prices
    → Screener (Pearson |ρ|>0.55 → Engle-Granger ADF p<0.10 → Johansen rank → OU params → entry_z)
    → Kalman Filter (2-state, β(t) updated every bar — no look-ahead drift)
    → Regime Classifier (Hurst + autocorr + vol ratio + |z| → XGBoost GBM)
    → Pair Trader (entry |z|>entry_z + regime==1, exit |z|<0.5 or |z|>4.0 stop or t>60)
    → Portfolio (greedy selection, max 4 pairs, cross-spread corr < 0.65)
    → Analytics Dashboard (8-panel PNG)
```

**Key Parameters (from `config.py`):**
| Parameter | Value | Rationale |
|-----------|-------|-----------|
| Correlation threshold | 0.55 | Pre-filter to reduce computation |
| EG p-value | 0.10 | Relaxed threshold for more pairs |
| Half-life range | 1–120 days | Practical holding window |
| KF process noise δ | 5e-5 | Tight tracking of β(t) |
| KF observation noise R | 0.001 | Balance responsiveness vs noise |
| Entry z | Optimal (Elliott 1994) ≈1.5–3.0 | Clipped to practical range |
| Exit z | 0.50 | Mean reversion target |
| Stop z | 4.00 | Structural break assumption |
| Max holding | 60 bars | Time stop avoids indefinite exposure |
| Max pairs | 4 | Concentration limit |
| Max cross-corr | 0.65 | Avoid correlated drawdowns |
| Transaction cost | 0.02% one-way per leg | Realistic for liquid pairs |

**Key Technical Insight:** The Kalman Filter hedge ratio is the critical differentiator. Static OLS β breaks in real markets due to regime shifts and carry changes. KF updates β every single bar with process noise δ=5e-5. This is essential for any live stat-arb system.

**Reference:** Pole (2007), Elliott (1994), Engle & Granger (1987), Johansen (1988).

**Action for us:** Extract the Kalman Filter screener pattern. The current Joel trading stack (backtrader + ccxt) has no KF implementation. Adding this to `scripts/` would be a major upgrade.

### 1.3 HIGH-SIGNAL FIND: Order Flow Imbalance (OFI) Microstructure Tool (Rust)
**Repo:** `jamestjw/ferrum-flow` | Rust | Apr 28, 2026
**URL:** https://github.com/jamestjw/ferrum-flow

**What it does:** Rust CLI that reads trade prints + order book snapshots from Alpaca IEX feed, computes OFI/NOFI/GOFI metrics, and proposes short-term actions (EnterLong/EnterShort/ConfirmLongAtVwap/ReverseShort/NoTrade).

**Metrics:**
- **OFI** (Order Flow Imbalance): Σ signed volume across window. Positive = aggressive buying pressure.
- **NOFI** (Normalized OFI): OFI / total volume. Comparable across liquid/illiquid names. Range [-1, 1].
- **GOFI** (Generalized OFI): Book depth aggregation. Bid-size increases minus ask-size increases across N levels.
- **VWAP:** Volume-weighted average trade price.
- **Linear Price Impact:** `lambda * OFI` with configurable λ.

**Signal Rules:**
| Signal | Condition | Execution Posture |
|--------|-----------|-------------------|
| EnterLong | NOFI > momentum threshold (0.20) | Aggressive |
| EnterShort | NOFI < -momentum threshold | Passive |
| ConfirmLongAtVwap | Price near VWAP + NOFI strongly positive | Aggressive |
| ReverseShort | Strong buy imbalance but flat price (absorption heuristic) | Aggressive |
| NoTrade | None of above met | Neutral |

**Watch Mode:** Continuously polls Alpaca on trailing window (default 300s, data-delayed 900s for free plans). 5-second poll interval.

**Action for us:** This is a clean Rust microstructure scaffold. Joel's stack is Python-based (ccxt + backtrader), but the OFI logic could be ported to a Python module using Alpaca's trade API. The absorption reversal heuristic (buy pressure + flat price = hidden sell liquidity) is a non-obvious pattern worth testing.

### 1.4 HIGH-SIGNAL FIND: Funding Rate Arbitrage Bots (3 variants found)

**A. BrokeBot — TRON Funding Rate Arb (Most Complete)**
**Repo:** `GareBear99/BrokeBot` ⭐2 | Updated May 4, 2026 (TODAY)
**URL:** https://github.com/GareBear99/BrokeBot

**Strategy:** Captures funding payments when rates are extreme (≥0.08%), while keeping price risk controlled via ATR-sized stops.
**Target:** $10–$20 accounts, 2-5x leverage, USDT-TRC20 on Binance Futures.

**Entry Filters (ALL must be true):**
1. Funding extreme: `abs(funding_rate) >= 0.08%`
2. Time window: 10–90 minutes before next funding
3. Liquidity: Spread ≤ 0.05%
4. Volatility calm: ATR(15m) below 7-day median, no 3x volume spikes in last 10 min
5. Trend filter: No strong opposing momentum (EMA20/50 crossover with momentum)

**Position Direction:**
- Funding > 0 → Longs pay shorts → **Open SHORT** (receive funding)
- Funding < 0 → Shorts pay longs → **Open LONG** (receive funding)

**Risk Management (very detailed):**
- Position sizing: Risk 1% per trade based on stop distance
- Stop-loss: 0.8x ATR(15m)
- Take-profit: 0.4x ATR (mean reversion)
- Max 2 entries per funding interval, 4 per day
- Daily loss halt: -2%
- Consecutive loss halt: 2 losses
- Drawdown halt: -15%
- Emergency exits: Spread > 0.12% or rapid price move > 1.2x ATR in 5 min

**Why this is gold:** The risk manager (`risk_manager.py`) is production-grade — state persistence, JSONL logging, halt flags with multiple independent triggers, position size calculation, emergency exit conditions. This is a reference implementation for ANY bot.
